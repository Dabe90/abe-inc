Architecture pattern

The same stack we shipped for Prayer City Social — proven in production, not slideware.

  1. 1Compose — user or system triggers a workflow with context (content, form data, roster row).
  2. 2Agent layer — prompt pipelines generate variants, classify intent, or extract structured data.
  3. 3Review — dashboard approval step before external actions.
  4. 4Execute — queue workers publish, send, or sync via OAuth APIs with retries.
