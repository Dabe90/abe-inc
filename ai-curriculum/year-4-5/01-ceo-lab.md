## Memo templates (write these for real)

### Memo A — Train vs buy
“Will we train a base model in the next 24 months?”  
Include: cost, talent, data, differentiation, opportunity cost.

### Memo B — Board / advisor update
Metrics · Wins · Risks · Asks.

### Memo C — Incident tabletop
Model leaked PII or sent bad email — timeline for 0–24h.

| Signal | How you test |
|--------|----------------|
| Ships | Take-home: fine-tune + eval report |
| Systems | Explain KV cache / quantization |
| Taste | Critique a bad eval |
| Communication | Write a model card |
| Safety | Prompt-injection fix design |

## Capital literacy

- Bootstrap with services (Abe Stack) → fund R&D  
- Raise only when a **product wedge** is clear  
- Never raise on “we’ll build K3 alone”
